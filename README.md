# Тест-кейсы проверки функциональности сервиса WordPress REST API  
   
## Тест-кейс 1:   
1.1 Создание нового поста через (POST /wp/v2/posts);  
1.2 Получение поста (GET /wp/v2/posts/<id>);  
1.3 Обновление поста (POST /wp/v2/posts/<id>);  
1.4 Удаление поста (DELETE /wp/v2/posts/<id>);  
1.5 Попытаться создать новый пост, негативный сценарий (несуществующий пользователь); 
1.6 Попытаться получить пост, негативный сценарий (несуществующий <id>) 
  
### Цель:  
Создать пост. Из ответа получить <id> созданного поста. Получить пост по <id>. Обновить пост по <id>.  Удалить пост по <id>.  
  
### Тестовые данные:  
 Заголовок и контент указаны в запросе:    
   - title: "testTitle"    
   - content: "testContent"    
   - status: "publish".    
 
---  
   
### Предварительные условия:  
   
1. На сервере настроен и запущен WordPress REST API.    
2. API-ключ (токен) для авторизации получен, и у пользователя (связанного с данным токеном) есть право на создание записей.    
3. Сформирован URL:  
http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fposts.    
   
---  
   
### Шаги воспроизведения:  
  
### 1.1 Тест создания поста :  
   
1. Выполнить POST-запрос с использованием `curl` по следующему адресу:    
   http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fposts   
2. Добавить в запрос headers:    
   Authorization: Bearer YOUR_TOKEN — обязательный параметр для подтверждения прав доступа через токен.    
3. Указать данные для нового поста в параметрах запроса:    
   - Поле `title="testTitle"` — задаёт заголовок поста;    
   - Поле `content="testContent"` — задаёт содержимое поста;  
   - Поле `status="publish"` — публикует пост сразу после создания;  
4. Пример запроса:  
curl --location 'http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fposts' \  
--header 'Authorization: ••••••' \  
--form 'title="testTitle"' \  
--form 'content="testContent"' \  
--form 'status="publish"'  
4. Выполнить запрос.    
5. Убедиться, что сервер возвращает успешный ответ.    
6. Передать <id> в следующие тесты.  
   
---  
   
### Ожидаемый результат:  
   
1. Сервер возвращает HTTP-статус: `201 Created`, если запись успешно создана.  
2. Ответ в формате JSON содержит <id>.  
  
---  
  
### 1.2 Тест получения поста по <id> :  
   
1. Выполнить POST-запрос с использованием `curl` по следующему адресу:    
   http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fposts/<id>  
2. Добавить в запрос headers:    
   Authorization: Bearer YOUR_TOKEN — обязательный параметр для подтверждения прав доступа через токен.    
3. Пример запроса:  
curl --location 'http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fposts%2F <id> '  
4. Выполнить запрос.    
5. Убедиться, что сервер возвращает успешный ответ.

Сервер возвращает HTTP-статус: `200 ОК`, если запись существует.

### 3.6 Тест обновления пользователя “ME”:   
1. Выполнить POST-запрос с использованием `curl` по следующему адресу:   
   http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fusers/me   
  
2. Добавить в запрос headers:   
   Authorization: Bearer YOUR_TOKEN — обязательный параметр для подтверждения прав доступа через токен.   
  
3. Указать данные для обновления пользователя в параметрах запроса:   
   - Поле `username="testName.updated"` — задаёт новое имя пользователя;   
  
4. Пример запроса:   
   curl --location 'http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fusers%2Fme' \ 
   --header 'Authorization: ••••••' \ 
   --form 'username="testName.updated"' 
5. Выполнить запрос.   
6. Убедиться, что сервер возвращает успешный ответ.   
  
---   
  
### Ожидаемый результат:   
Сервер возвращает HTTP-статус: `200 OK`, если пользователь успешно обновлён.

### 3.7 Тест создания пользователя, негативный сценарий (несуществующий токен):   
1. Выполнить POST-запрос с использованием `curl` по следующему адресу:   
   http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fusers   
  
2. Добавить в запрос headers:   
   Authorization: Bearer YOUR_INVALID_TOKEN — токен должен быть сформирован по неверным данным пользователя.   
  
3. Указать данные для нового пользователя в параметрах запроса:   
   - Поле `username="testName.lastName"` — задаёт имя пользователя;   
   - Поле `email="testmail@gmail.com"` — задаёт адрес электронной почты;   
   - Поле `password="123Test"` — задаёт пароль;   
  
4. Пример запроса:   
   curl --location 'http://localhost:8000/index.php?rest_route=%2Fwp%2Fv2%2Fusers' \ 
   --header 'Authorization: ••••••' \ 
   --form 'username="testName.lastName"' \ 
   --form 'email="testmail@gmail.com"' \ 
   --form 'password="123Test"' 
5. Выполнить запрос.   
6. Убедиться, что сервер возвращает негативный ответ.   
  
---   
  
### Ожидаемый результат:   
1. Сервер возвращает HTTP-статус: `401 Unauthorized`, если пользователь не авторизован.   
2. Ответ в формате JSON содержит сообщение об ошибке.
